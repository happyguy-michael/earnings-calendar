# SkipLink

**Date:** 2026-03-27
**Type:** Accessibility Component
**Status:** Shipped ✅

## Summary

Added a SkipLink component that provides keyboard users a way to bypass header navigation and jump directly to main content. This is a fundamental accessibility feature for screen reader users and keyboard-only navigation.

## Inspiration

- WCAG 2.1 Success Criterion 2.4.1 (Bypass Blocks)
- Apple's Human Interface Guidelines for accessibility
- GitHub, Linear, and other modern apps with skip links

## Features

1. **Invisible Until Focused:**
   - Completely hidden from visual display
   - Slides in from top when user presses Tab
   - Spring-physics animation (bouncy)

2. **Premium Visual Design:**
   - Gradient background (blue → purple)
   - Frosted glass effect with backdrop blur
   - Down arrow icon with bouncing animation
   - "Enter" keyboard hint badge

3. **Focus Management:**
   - Sets focus to main content after click
   - Temporary tabindex for screen readers
   - Proper focus ring styling

4. **Variants:**
   - `SkipLink`: Single target (main content)
   - `SkipLinks`: Multi-target navigation bar with numbered shortcuts

5. **Accessibility:**
   - Proper ARIA labels
   - Respects prefers-reduced-motion
   - Dark/light mode support
   - Keyboard-only navigation

## Props

```typescript
interface SkipLinkProps {
  targetId?: string;      // ID to skip to (default: 'main-content')
  label?: string;         // Custom label text
  offsetTop?: number;     // Offset for sticky headers (default: 120px)
  className?: string;     // Additional styles
}
```

## Usage

```tsx
// In page.tsx (first focusable element)
<SkipLink targetId="main-content" offsetTop={140} />

// Target element
<main id="main-content">
```

## Technical Notes

- Uses CSS custom property `--spring-bouncy` for physics-based animation
- Offset compensates for the sticky header (140px)
- Focus is set with `preventScroll: true` to avoid double-scroll
- Tabindex is removed on blur to maintain natural tab order

## Why This Matters

Skip links are essential for:
- Screen reader users navigating complex pages
- Keyboard-only users who can't mouse-click
- Power users who want fast navigation
- WCAG compliance (Level A)

Without skip links, keyboard users must Tab through every header element to reach main content—on this page, that's 50+ elements.
